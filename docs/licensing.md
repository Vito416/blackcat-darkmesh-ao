# Licensing proposal

Goal: auditovatelný, široce adoptovatelný software, ale příjem z provozu/registrace.

## Navržený model
- **Apache-2.0 pro zdrojový kód**: minimální bariéra, snadný audit, kompatibilita s integrátory.
- **Trademark policy**: název/branding zůstává chráněn; forky bez brandu.
- **Registrace/verified listing**: jednorázový (malý) poplatek na Arweave pro zápis do veřejného registru + nárok na podporu/SLA.
- **Komerční nabídka**: hostovaný cloud, SLA a support, volitelné enterprise doplňky (HA, zálohy, compliance).

## Proč Apache + registrace
- Umožňuje maximální adopci a audity (bez copyleftu, takže integrátoři se nebojí).
- Monetizace není přes licenci, ale přes služby/registraci (on‑chain, transparentní).
- Otevřený kód posiluje důvěru v bezpečnost (PIP offline, anti‑cenzura), registry poplatek pokrývá provoz a support.

## Další kroky (pokud přejdeme na Apache)
1. Nahradit `LICENSE` za Apache-2.0 text, přidat `NOTICE`.
2. Přidat `TRADEMARKS.md` s pravidly pro název/logo.
3. Do README uvést: kód pod Apache-2.0, registrace poplatek na Arweave pro oficiální listing.
4. Připravit ceník/SLA (Starter/Pro/Enterprise) a zveřejnit kontaktní kanál.

_Tento dokument je doporučení; stávající stav je v kořenovém souboru `LICENSE`._

# Blackcat Mesh NEXUS Licensing System Notice

Status: tracked copy of the workspace-level system notice
Date: 2026-03-19

This document mirrors the workspace-level `LICENSING_SYSTEM_NOTICE.md` placed in `BLACKCAT_MESH_NEXUS/`.

Its purpose is to keep a version-controlled copy of the same notice inside a tracked repository, because the `BLACKCAT_MESH_NEXUS` root is not itself a git repository.

## Purpose

This notice makes the current licensing intent explicit:

- every official repository currently present under the top-level `BLACKCAT_MESH_NEXUS` workspace is intended to be interpreted as part of one `Covered System`;
- current internal top-level workspace directories are also presumed part of that same Covered System unless a signed public Founder Notice expressly excludes them;
- repository or directory separation is used for maintenance, safety, auditability, delivery, and architectural clarity;
- and that separation does not by itself create separate unavoidable founder-fee or steward/development-fee events for the same ordinary covered deployment.

## Current interpretation

The intended current interpretation is:

- the repositories and internal top-level workspace components listed above form one Blackcat Covered System;
- repository or directory separation does not by itself create separate founder-fee or steward/development-fee events;
- ordinary deployment classes should be interpreted on an integrated-system basis unless a later published fee policy clearly and non-pretextually defines a genuinely different coverage class;
- and future internal packages, repos, feeds, control services, or maintained replacements must remain subject to the same accessibility, anti-capture, and anti-decay guardrails as the rest of the Covered System.

## Relationship to repository-local LICENSE files

This system notice is interpretive and organizational.

The repository-local `LICENSE` files remain the direct license notices for the repositories they live in. Those `LICENSE` files should now use the same active BFNL 1.0 text and should be read together with the canonical bundle above.

## Future additions and exclusions

Future official repositories, internal packages, or internal top-level workspace directories added under `BLACKCAT_MESH_NEXUS` are presumed to join the same Covered System unless a signed public Founder Notice expressly states otherwise.

Exclusions or carve-outs should be explicit, public, and authenticated. Silence should not be interpreted as an implicit carve-out.

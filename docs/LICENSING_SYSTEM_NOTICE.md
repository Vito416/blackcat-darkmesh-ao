# Blackcat Mesh Nexus Licensing System Notice

Status: tracked copy of the workspace-level system notice
Date: 2026-03-19

This document mirrors the workspace-level `LICENSING_SYSTEM_NOTICE.md` placed in `BLACKCAT_MESH_NEXUS/`.

Its purpose is to keep a version-controlled copy of the same notice inside a tracked repository, because the `BLACKCAT_MESH_NEXUS` root is not itself a git repository.

## Purpose

This notice exists to make one thing explicit:

- the main official Blackcat repositories are intended to be interpreted as parts of one `Covered System`,
- not as unrelated products whose repository separation automatically creates separate unavoidable founder-fee or steward/development-fee events.

Repository separation is used for maintenance, safety, auditability, delivery, and architectural clarity.

## Repositories included in this umbrella transition step

The following repositories are included in the present umbrella-licensing transition step:

- `blackcat-darkmesh-ao`
- `blackcat-darkmesh-write`
- `blackcat-core`
- `blackcat-kernel-contracts`
- `blackcat-darkmesh-gateway`
- `blackcat-darkmesh-web`

Each of those repositories now contains a matching `LICENSE` file using the same `Blackcat Covered System Transitional License Notice`.

## Current interpretation

The intended current interpretation for the repositories listed above is:

- they are official components of one Blackcat Covered System;
- repository separation does not by itself create a separate founder fee or future steward/development fee for each repository;
- and ordinary deployment classes should be interpreted on an integrated system basis unless a later formally adopted fee policy clearly and non-pretextually defines a genuinely different coverage class.

## Relationship to repository-local LICENSE files

This system notice is interpretive and organizational.

The repository-local `LICENSE` files remain the direct license notices for the repositories they live in. Those `LICENSE` files now use the same transitional covered-system framework and point back to the founder-fee bundle for the planned future umbrella model.

## Future additions

Other sibling repositories in `BLACKCAT_MESH_NEXUS` may later be added to the same covered-system umbrella by:

- updating their repository-local `LICENSE` file,
- updating their README or public licensing notice,
- or adding them through a signed public notice in the future umbrella licensing system.
